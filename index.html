Todo lo aprendido en Git y GitHub

Thomas Gomez | Rodrigo Stepaniuk

¿Qué es Git & GitHub?

Git:Es un sistema de control de versiones, es decir, un sistema que registra los cambios en archivos. Esto nos permite recuperar versiones anteriores si incurrimos en errores en el códgio o si realizamos cambios que luego no queremos implementar en el proyecto.

GitHub:Escencialmente es un repositorio remoto para compartir y guardar proyectos utilizando el sitema de control de versiones Git.

Comandos mas utilizados

git init: Inciar un repositorio, debes ejecutarlo en la carpeta donde esten los archivos que pertenezcan al repo.

git checkout -b [branch]: Utilizamos este comando para crear una nueva rama y posicionarnos en ella.

git push -u origin [branch]: Con este comando, confirmamos que nuestra rama fue configurada para trackear una rama remota

git add [archivo]: agregar archivo a la etapa de stagging (memoria). Este archivo quedara en espera de ser agregado al repositorio remoto o ser eliminado.

git rm --cached [archivo]: Lo utilizamos para borrar los archivos agregados (con el comando 'add') la etapa de stagging

git status: Muestra los commits con sus respectivos mensajes y los cambios que se hicieron en el repositorio.

git commit -m 'mensaje': Guarda en el repositorio todos los cambios que hicismos; debemos escribir un mensaje que describa respectivos cambios.

git diff: Muestra los cambios entre los distintos commits.

git log:Muestra los ID's de los commits.

git pull: Con este comando traemos los ultimos cambios del repositorio remoto.

git push: Enviamos los cambios de nuestro último commit al repositorio remoto.

git reset:Permite volver a versiones anteriores borrando los cambios que hicimos luego de determinado commit. Reset --hard, reestablece el commit a 'staging' y también el directorio de trabajo. Reset --soft, reinicia el HEAD para el commit, NO reinicia el arbol de trabajo.

Merge y fusión de ramas

¿Qué es un Merge?Un merge es una fusión de una rama en la que se realizaron distintas modificaciones de los archivos, con otra rama a la que queremos agregarle esos nuevos cambios.

git merge [rama ubicado] [rama que mergeamos]:Agregamos a la rama en la que estamos parados, los cambios que se hicieron en otra rama.

git branch:Nos muestra las ramas creadas hasta el moento.

Alias & Tags

git log --all --graph --decorate --oneline Este comando nos muestra todas las ramas de commits que fuimos realizando.

git tag -a v0.1 -m 'version 0.1' [hash commit] Creamos un tag basado en determinado commit.

git tag Vemos los tags creados

git push origin --tagsEnviar los tags creados al repo remoto.

git tag -d [tag] Borramos el tag en el repositorio local.

git push origin :refs/tags/[tag]Borramos el tag en el repositorio remoto.

Crear y configurar una SSH Key

¿Por qué crear y configurar una SSH Key?Por cuestiones de seguridad, es importante configurar una SSH para utilizar Git y GitHub, y así evitar passwords craking y proteger todos los archivos que estén alojados GitHub, tanto proyectos personales propios o proyectos profesionales para terceros.

¿Cómo funciona una SSH Key?En nuestro repo local crea una llave pública y una privada (esto se crea para cada repositorio), la llave pública es enviada a GitHub y luego GitHub, envía su propia llave pública (cifrada con tú llave pública) a nuestro repo local, es así que se configura una conexión de doble camino SSH, mucho más segura que la HTTPS,gracias a esto ya no debemos utilizar contraseña.

git config --global user.email [mail]:Nos permite configurar nuestro nombre de email, este debe matchear con el de nuestra cuenta de GitHub.

git config --global user.name:Configuraremos nuestro nombre de usuario, es el nombre que se muestra en autores cuando vemos los registros de commits.

ssh-keygen -t rsa -b 4096 -C '[email]:'Con este comando creamos la llave SSH, definimos el algoritmo para crear la llave, que tan compleja será la llave y el correo al que estará conectada la llave.

eval $(ssh-agent - s):Comprobamos el proceso de la creación de llave.

ssh-add [locación del la llave privada]:Agregamos la llave que creamos al agente de autenticación.

Configurar una SSH Key con GitHub:Copiamos nuestra llave pública, vamos a GitHub > Settings > SSH and GPG keys; ponemos un título y creamos la llave.

git remote set-url [SSH]Ahora configuramos la conexion SSH del repositorio remoto al que queremos conectarnos.

git remote -v:Vemos las direcciones de origin.
